3. Mobile First vs Desktop First

Two approaches to writing responsive CSS. Mobile-first is generally recommended.

üì± Mobile First

Start with mobile styles, enhance for larger screens.

.box { width: 100%; }

@media (min-width: 768px) {
  .box { width: 50%; }
}

üñ•Ô∏è Desktop First

Start with desktop styles, reduce for smaller screens.

@media (max-width: 767px) {
  .box { width: 100%; }
}

4. Responsive Typography

Text should be readable at all screen sizes. Here are different approaches:

font-size: 24px; (Fixed)

This text size stays the same at all viewport widths.

font-size: 4vw; (Viewport Width)

This text scales with the viewport width!

font-size: clamp(16px, 4vw, 36px); (Clamp - Best Practice)

This text scales but has min/max limits for readability!

Using media queries to change font-size

üí° Clamp() is the best approach!
clamp(min, preferred, max) - The text will be the preferred size (usually viewport-relative) but never smaller than min or larger than max.

10. Responsive Units

Modern CSS offers various responsive units beyond pixels and percentages.

Viewport Units

width: 50vw (50% of viewport width)
height: 20vh (20% of viewport height)

vmin - Based on Smaller Viewport Dimension

30vmin √ó 30vmin

Percentage

width: 75%
üí° Unit Reference:
vw - 1% of viewport width | vh - 1% of viewport height
vmin - 1% of smaller dimension | vmax - 1% of larger dimension
% - Relative to parent element
